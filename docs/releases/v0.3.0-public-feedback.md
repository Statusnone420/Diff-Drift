# v0.3.0 — Public Feedback Release

This is an early public feedback release. The point of this release is to be judged, not admired.

Diff Drift is a local tool for reviewing the code drift an AI coding agent leaves behind. You pick a baseline you trust — `HEAD`, your last review's trust point, or the merge-base with `main` — and it shows you what structurally changed since then, flags drift that looks security-relevant, and tracks what you've already reviewed so the next agent run only reopens what changed again.

## What it is not

Diff Drift is not a full static analyzer, and a clean run is not proof that code is safe. The flags are heuristic prompts for a human reviewer, not verdicts. It runs entirely locally: no LLM calls, no telemetry, no repository upload — see [Privacy and Data Flow](../wiki/Privacy-and-Data-Flow.md) if you'd rather verify that than take our word for it.

## What we want feedback on

1. **Install friction.** Did you get from download to a working app? Where did it hurt? ([install problem template](https://github.com/Statusnone420/Diff-Drift/issues/new?template=install-problem.yml))
2. **Trust-point clarity.** Does "review changes since the last state you trusted" make sense in the UI, or did the concept lose you? ([confusing output template](https://github.com/Statusnone420/Diff-Drift/issues/new?template=confusing-output.yml))
3. **Noisy flags.** Which flags fired on code that's fine in your codebase? ([noisy flag template](https://github.com/Statusnone420/Diff-Drift/issues/new?template=noisy-flag.yml))
4. **The core question:** would you use this before trusting code changed by an AI coding agent? Why or why not? ([tried it template](https://github.com/Statusnone420/Diff-Drift/issues/new?template=tried-diff-drift.yml))

## Known limits

- Windows-first. macOS is experimental and unsigned.
- The installer is unsigned, so SmartScreen will warn on first run. Checksums ship with each release; see [Release and Platform Support](../wiki/Release-and-Platform-Support.md).
- Analysis covers TypeScript, TSX, JavaScript, and JSX, plus `package.json` dependency drift. Other languages show up as changed files, not analyzed drift.
- Flags are heuristics. Expect some noise; that's part of what this release is measuring.
- Not a vulnerability guarantee. Run a real SAST tool alongside if you need one.

## Try it

Download the installer from [Releases](https://github.com/Statusnone420/Diff-Drift/releases) and verify against `SHA256SUMS.txt`, or use the CLI:

```bash
diff-drift check . --baseline merge-base --md > diff-drift-report.md
```

Full usage in the [User Guide](../wiki/User-Guide.md).
