# Feedback Triage

Every piece of feedback from the public feedback release gets one bucket and one row in the log. The point is to spot repeats, not to react to individual comments. Ten strangers should not become ten product directions.

## Buckets

| Bucket | Sounds like | Priority |
| --- | --- | --- |
| Install failure | "SmartScreen scared me off", "installer failed", "couldn't find the CLI" | High |
| Positioning confusion | "I don't get why this exists" | High |
| Core-loop confusion | "What is a trust point?" | High |
| Noise / false positive | "This flag is dumb in my codebase" | High if repeated |
| Missing platform/language | "I need macOS / Python / Go" | Track, don't chase |
| Aesthetic criticism | "UI is ugly" | Low unless it blocks comprehension |
| Bad-faith dunking | "This is pointless garbage" | Ignore unless it names a real failure |
